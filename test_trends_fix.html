Тест исправления графика трендов

Описание проблемы

В отчете по оценке объекта недвижимости график трендов показывал значения, отличные от тех, что отображались в таблице "Детальные данные трендов".

Например, для июля 2025 года:

  • В таблице: ₺50 933,66
  • На графике: ₺48 035,00

Причина проблемы

График использовал оригинальные данные из базы данных, в то время как таблица отображала рассчитанные значения с учетом процентов изменения.

// Было: initializeTrendsChart(trends); // Оригинальные данные // Стало: const trendsWithHistoricalPrices = await prepareTrendsWithHistoricalPrices(trends); initializeTrendsChart(trendsWithHistoricalPrices); // Рассчитанные данные

Внесенные изменения

  1. ✅ Обновлен вызов initializeTrendsChart для использования подготовленных данных
  2. ✅ Обновлено сохранение данных в window.currentTrendsData
  3. ✅ Данные теперь подготавливаются при загрузке трендов
  4. ✅ Функция переключения типа графика уже использует правильные данные

Результат

Теперь график трендов будет отображать те же значения, что и в таблице "Детальные данные трендов", поскольку оба компонента используют одни и те же рассчитанные данные.

Для проверки исправления необходимо:

  1. Открыть отчет по оценке объекта недвижимости
  2. Сравнить значения в таблице "Детальные данные трендов" со значениями на графике
  3. Убедиться, что значения совпадают
